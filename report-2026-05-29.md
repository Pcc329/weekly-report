## 一、本週整體進度

本週做了五件事：

**① 系統找方案的方式壞了，我們找到原因並修好了**
之前用 AI 搜尋「餐飲業想解決人力短缺」，系統其實沒有真的在篩選——每次都回傳全部 2,169 筆，只是假裝有搜尋。本週改掉這個邏輯，讓 AI 真的去理解使用者的需求再過濾。改完之後重跑測試，搜尋結果從「每次都 2,169 筆」變成平均 105 筆，而且不同問題真的會得到不同答案。

**② 測試出哪些類型的方案，資料庫根本沒有**
用 100 個「產業 × 需求」情境測試，發現有 22 個情境搜尋結果是 0 筆。最嚴重的是「提升辦公室效率」（9 個產業都找不到）和「吸引人才」（5 個產業找不到）。這代表資料庫在這兩類需求上幾乎空白，需要補充。

**③ 資料庫新增 153 筆，3 個搜尋異常全修復**
從臺灣雲市集爬蟲抓進 153 筆新方案，總筆數來到 2,322 筆。原本有 3 個情境會把全庫 2,322 筆都丟出來——問題是 AI 看不懂「接觸新客戶」「改善客戶體驗」這類模糊詞，不知道要對應什麼關鍵字。本週補充了 8 組需求對照規則並修正，驗證通過，3 個異常情境全部正常。

**④ 整理 7 個資料來源的關聯，讓長官看得懂**
資料庫收錄了 7 個不同政府單位的方案，本週整理清楚它們的定位和關聯：探索層（找方案）→ 補助層（申請補助）→ 採購層（政府採購）。並新增「資料來源說明」獨立頁面，在首頁右上角加入入口按鈕，直接可以分享給長官看。

**⑤ 建立報告中心，工作歷程透明化**
週報、ETL 資料分析報告統一放在報告中心，歷史版本自動累積，主管隨時可以看到完整工作軌跡。收到共構平台組(軟採網)的資料後，也完成第一份 ETL 分析報告（27,092 筆 → 去重後 21,948 筆）。

## 五、Round 03 缺口分析

> **白話說明**：我們模擬中小企業老闆來查詢，例如「餐飲業想要提升辦公室效率」，結果是 0 筆——代表資料庫裡根本沒有對應的工具。這就像圖書館裡沒有這本書，不是讀者的問題，是我們的書不夠。

### 系統性缺口（需優先補充）

| 需求 | 0筆次數 | 缺口產業 |
|---|---|---|
| **提升辦公室效率** | **9 / 10** | 餐飲、製造、零售、醫療、建築營造、金融、教育、旅遊住宿、電商 |
| **吸引人才** | **5 / 10** | 零售、醫療、教育、旅遊住宿、電商 |
| 加強資訊安全 | 2 / 10 | 物流、教育 |
| 數位轉型入門 | 2 / 10 | 醫療、教育 |

> **結論**：「提升辦公室效率」（OA 工具類）和「吸引人才」（HR 科技類）是資料庫最大系統性缺口，需優先補充。

## 六、資料來源整合分析

> **為什麼要整合這些來源？**
> 資料庫收錄了 7 個不同政府單位和平台的方案，長官可能會問：「這些方案之間有什麼關係？同一家公司的方案在不同地方出現，算一個還是多個？企業可以同時用到幾個？」這一節就是回答這個問題。

### 現有資料來源總覽

| 來源（program_type） | 主管機關 | 筆數 | 補助類型 | 主要使用對象 |
|---|---|---|---|---|
| 臺灣雲市集 | 數位部 | 1,231 | 採購平台（無直補）| 中小企業、政府機關 |
| 雲市集工業館 | 數位部/工業局 | 329 | 採購平台（製造業專區）| 製造業 |
| SMEI平台 | 中企署 | 279 | 政府補助方案 | 中小企業 |
| 領域型調查(人工搜查) | III 自有 | 133 | 無補助，純資訊 | 五大領域業者 |
| 新創嚴選網 | 國發會 | 132 | 新創認證 | 想導入創新方案的企業 |
| 商業服務業專區 | 商業署 | 117 | 政府補助方案 | 餐飲、零售、服務業 |
| 政府軟體採購網 | 共構平台組(軟採網) | 100 | 政府採購（已完成）| 政府機關、公立學校 |
| **合計** | | **2,322** | | |

### 這些來源的關係，用一句話說

7 個來源可以分成三層，對企業主來說是「**找方案 → 看補助 → 採購**」的完整路徑：

```
【探索層】找到適合的方案
  臺灣雲市集 ─────┐
  雲市集工業館 ───┤─→ 企業看到方案，評估要不要買
  領域型調查 ─────┘

【補助層】有政府補助可以幫你出錢
  SMEI平台 ────────┐
  商業服務業專區 ──┤─→ 企業確認可以申請哪些補助
  新創嚴選網 ──────┘

【採購層】大型組織直接走政府採購
  政府軟體採購網 ──→ 政府機關 / 學校 走這條路（與中小企業路徑不同）
```

### 三條「組合路徑」給長官參考

| 路徑 | 情境 | 對應來源 |
|---|---|---|
| **路徑 A** | 中小企業最常走：申請補助買雲端工具 | SMEI平台 + 臺灣雲市集 |
| **路徑 B** | 製造業數位轉型：補助購買工業 4.0 方案 | SMEI平台 + 雲市集工業館 |
| **路徑 C** | 商服業導入 AI：補助購買新創工具 | 商業服務業專區 + 新創嚴選網 |

### 一個需要特別說明的問題

**政府軟體採購網（軟採網）和其他來源定位差很大。** 軟採網是給政府機關採購大型授權用的（price 中位數 NT$ 124,744），不是給中小企業數位轉型用的。建議在搜尋介面上**獨立顯示**，避免和其他方案混在一起讓中小企業老闆看了一頭霧水。
