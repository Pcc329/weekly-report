## 五、新創業者自動識別機制

### 設計邏輯

以**成立日期起算 8 年內**為新創標準，取代原本的人工標記方式。

```
established_date 距今 ≤ 8 年 → is_startup = 新創
established_date 距今 > 8 年 → is_startup = 非新創
無成立日期 → 保持原狀，待人工補齊
```

### 本次批次更新結果

| 項目             | 數字        |
| -------------- | --------- |
| 重算母體           | 848 家     |
| 移除誤標（超過 8 年）   | 26 家      |
| 新增標記（8 年內但未標記） | 11 家      |
| 無日期保持原狀        | 189 家     |
| 更新後新創總數        | **219 家** |

### 後續維護計畫

| 階段      | 方式                   | 說明                      |
| ------- | -------------------- | ----------------------- |
| 短期（已完成） | 批次腳本手動跑              | 每次新業者匯入後執行一次            |
| 中期（本週）  | Airtable Formula 欄位  | 登入 Airtable 自動重算，作為視覺警示 |
| 長期（下個月） | Vercel Cron Job 每日排程 | 完全自動，無需人工介入             |

> 這個機制讓「今天是新創，明天可能就不是」成為可能。資料庫裡的每家公司，未來都會每天自動確認自己的新創狀態，不再依賴人工判斷。
